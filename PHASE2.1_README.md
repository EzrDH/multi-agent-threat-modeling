## 🚀 Setup Steps

Assuming kamu sudah punya Phase 1 MVP yang jalan:

### 1. Update Dependencies

Extract zip ini ke folder project-mu, lalu install dependencies baru:

```bash
cd multi-agent-threat-modeling
pip install -r requirements.txt
```

Dependencies baru yang akan terinstall:
- `chromadb` — vector database lokal
- `langchain-google-genai` — Gemini embeddings integration
- `tiktoken` — token counting
- `markdown` — parse ASVS markdown
- `requests` — download utilities

### 2. Verifikasi Konfigurasi

```bash
python -c "from src.config import print_config_summary; print_config_summary()"
```

Harus menampilkan:
- Embedding Model: `models/text-embedding-004`
- Knowledge Base Dir: `data/knowledge_base/`
- ChromaDB Dir: `data/knowledge_base/chroma_db`

### 3. Build Knowledge Base

**Penting:** Step ini akan memanggil Gemini API untuk embedding. Pastikan `GEMINI_API_KEY` di `.env` valid.

```bash
python scripts/build_knowledge_base.py
```

Apa yang terjadi:
1. Load ASVS markdown (V11.1 sampai V11.10, ~20 requirements)
2. Load NIST JSON (8 controls: SC-8, SC-12, SC-13, SC-17, SC-23, SC-28, IA-5, IA-7)
3. Chunk per requirement/control (~28 chunks total)
4. Embed via Gemini `text-embedding-004` (768-dim vectors)
5. Store di ChromaDB local (`data/knowledge_base/chroma_db/`)

**Waktu estimasi:** 2-5 menit (karena rate limit Gemini free tier 15 req/menit, ada delay 2 detik per batch).

### 4. Test Retrieval

```bash
python -m tests.test_kb_retrieval
```

Script akan test 5 threat dari Phase 1 MVP:
1. Weak Password Hashing (CWE-916) → harusnya retrieve V11.4.2 + IA-5
2. Insecure JWT Signing (CWE-347) → harusnya retrieve V11.8
3. Weak TLS (CWE-327) → harusnya retrieve V11.7 + SC-8
4. Predictable Token (CWE-640) → harusnya retrieve V11.5 + SC-23
5. Cleartext Communication (CWE-319) → harusnya retrieve SC-8 + V11.7

**Target accuracy: ≥60%** (matched expected keywords / total expected keywords).

## 🧰 Cara Pakai Knowledge Base Programatically

```python
from src.knowledge_base import KnowledgeBaseRetriever

# Initialize (otomatis konek ke ChromaDB yang sudah di-build)
retriever = KnowledgeBaseRetriever()

# Retrieve untuk threat tertentu
results = retriever.retrieve_for_threat(
    threat_title="Weak Password Hashing",
    threat_description="System uses MD5 to hash user passwords",
    cwe_id="CWE-916",
    top_k=5,
)

# Tampilkan hasil
for r in results:
    print(f"[{r['chunk_id']}] similarity={r['similarity']:.3f}")
    print(r['content'][:200])
    print()

# Atau format sebagai context untuk LLM prompt
context = retriever.format_as_context(results, max_chunks=3)
print(context)
```

## 🔧 Troubleshooting

### Error: `chromadb` not found
```bash
pip install chromadb --upgrade
```

### Error: Rate limit / 429 dari Gemini API
Script otomatis retry dengan delay. Tapi kalau persisten:
- Tunggu 1 menit, lalu re-run
- Atau gunakan Gemini Pro API key (kuota lebih besar)

### Error: ChromaDB collection sudah ada
Re-run dengan flag `--rebuild`:
```bash
python scripts/build_knowledge_base.py --rebuild
```

### Embedding terlalu lambat
Wajar karena free tier Gemini: 15 req/min × 28 chunks = ~2 menit minimum.
